# Coding Standards

## General Principles

- DRY, KISS, SOLID
- Clean Code practices

## Naming Conventions

- Variables: {e.g., camelCase}
- Functions: {e.g., camelCase}
- Classes: {e.g., PascalCase}
- Files: {e.g., kebab-case}

## Formatting

- Indentation: {e.g., 2 spaces}
- Line Length: {e.g., 80 characters}
- Braces: {e.g., K&R style}

## Comments & Documentation

- JSDoc for all exported functions and classes (TypeScript/JavaScript)
- Explain WHY, not WHAT — avoid comments that merely paraphrase the code; document intent, constraints, and decisions instead

## Testing

- Unit tests for all business logic
- Integration tests for API endpoints
